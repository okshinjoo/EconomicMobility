# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

"Empower" (Economic Mobility Project) is a free financial-education website aimed at first-generation, low-income, and immigrant youth. Next.js 16 (App Router) + React 19 + Tailwind CSS v4, all TypeScript. No backend yet — everything is static/client-side.

## Commands

```bash
npm run dev      # dev server at http://localhost:3000 (Turbopack)
npm run build    # production build
npm run start    # serve the production build
npm run lint     # ESLint (eslint-config-next: core-web-vitals + typescript)
```

There is no test framework configured. Verify changes by running `npm run dev` and checking the page, and `npm run build` to catch type errors (the build type-checks the whole tree).

## Architecture

Built routes: `/` (`app/page.tsx`), `/quiz`, `/about`, `/learn` (hub), `/learn/[topic]` (SSG per `TopicId`), `/learn/[topic]/[article]` (SSG, 112 article pages), `/glossary`, `/tools` (hub), and the calculators under it. Calculators are organized into **categories** (`lib/toolsRegistry.ts`): each category has a *main* calculator at `/tools/{budget,debt,savings,college}` plus focused *sub*-calculators nested under it (e.g. `/tools/debt/auto-loan`, `/tools/debt/mortgage`, `/tools/budget/paycheck`, `/tools/savings/compound`, `/tools/savings/retirement`, `/tools/savings/roth-ira`, `/tools/college/student-loan`). Every calc page shows `components/CalcSwitcher.tsx` (pill nav across the category; "soon" items in the registry render disabled). New sub-calc pages use `components/CalcPageShell.tsx`. To add a calculator: add it to `toolsRegistry.ts`, then build its page (it appears in the switcher + hub automatically). Invalid `[topic]`/`[article]` slugs `notFound()`. Also built: `/start-here` (onboarding + FAQ), `/resources` (curated external links + a per-state finder, see below), `/contact` (`components/ContactForm.tsx` — a no-backend `mailto` form to `help@economicmobilityproject.org`, the project inbox), `/tools/templates` (downloadable spreadsheet templates, see below), and `/blog` (the **"Ask Empower" Q&A** — anonymous question form + curated answered questions, see below). **Every header/footer nav link now resolves** — no 404s.

**Downloadable templates** (`/tools/templates`): three `.xlsx` files with live SUM/IFERROR formulas + brand styling (budget, debt payoff tracker, savings goal worksheet). The binaries live in `public/templates/` and are GENERATED by `scripts/build-templates.mjs` (ExcelJS, a devDependency) — run `npm run build:templates`, never hand-edit the `.xlsx`. The card metadata is in `lib/templates.ts` (shared by the page + a promo band on the `/tools` hub). To add one: add an entry to both `build-templates.mjs` and `lib/templates.ts`, then rerun the script.

**"Ask a Question"** (`/blog`): `components/AskQuestion.tsx` is an anonymous submission form (question + optional topic + optional email). It POSTs JSON to **Web3Forms** (`https://api.web3forms.com/submit`) gated on the `WEB3FORMS_ACCESS_KEY` constant — currently **empty**, so it runs in "preview mode" (confirms to the visitor but sends nowhere). To go live: get a free access key at web3forms.com (tied to `help@economicmobilityproject.org`), paste it into that constant, redeploy. (Submissions then land in that inbox. Swap the fetch for Formspree/Tally if preferred.) The answered Q&As shown below the form are curated content in `lib/communityQuestions.ts` (`CommunityQA[]`, topic-tagged) — add an entry when a good question comes in.

**Accessibility/mobile:** `globals.css` has global `:focus-visible` rings (amber, works on light + dark) and a `prefers-reduced-motion` block. Chart SVGs (`components/Charts.tsx`) are `aria-hidden` since their data is present as text in legends/result rows. All pages verified at 390px with zero horizontal overflow; the calculators' two-column layout stacks below `lg`, the article ToC is `xl`-only. (Capture mobile screenshots via Chrome **CDP `Emulation.setDeviceMetricsOverride`** — a plain `--window-size` screenshot does NOT emulate the mobile viewport and gives false overflow.)

**Calculators** (all `"use client"`, two-column inputs/sticky-results, localStorage-persisted, plain-language insight callouts): Budget Planner, Debt Payoff (`components/DebtCalculator.tsx` + `lib/debtPayoff.ts` — avalanche/snowball simulation; loan amortization via `loanPayment` matches standard calculators to the dollar; the "minimums only" baseline uses a *declining* revolving-card minimum (1% + interest, $25 floor) so credit-card payoff is realistic, while a debt with a `term` amortizes on its fixed schedule; blank minimums are estimated via `estimateMinPayment`), Savings Goal (`components/SavingsCalculator.tsx` + `lib/savingsGoal.ts` — by-deadline or by-amount with compound growth), College Cost (`components/CollegeCalculator.tsx` + `lib/collegeCost.ts` — unmet-need + loan repayment estimate). The three newer ones share presentational primitives + `usd`/`num`/`formatDuration` from `components/CalcUI.tsx`; each persists under its own `STORAGE_KEYS` entry. Data viz is `components/Charts.tsx` (dependency-free SVG): `Donut`, `TrendChart` (balance-over-time), and `Legend`, used across calculators. Two config-driven reusable calculators cover several tools each: `components/LoanCalculator.tsx` (auto/mortgage/student-loan — a `LoanConfig` of amount fields summed/subtracted into a principal, then `loanPayment`) and `components/GrowthCalculator.tsx` (compound/retirement — `growthSeries`, time by years or by ages). `components/RothIraCalculator.tsx` + `lib/rothIra.ts` is a dedicated Roth IRA calc (after-tax contributions growing tax-free, with a side-by-side comparison vs. a taxable account that loses return to a "tax drag"; the 2026 contribution limits `ROTH_LIMIT`/`ROTH_LIMIT_50PLUS` live in `lib/rothIra.ts` — update yearly). `components/PaycheckCalculator.tsx` reuses the `estimateTaxes` engine. The Budget Planner predates `CalcUI` and keeps its own copies.

The **budget calculator** (`components/BudgetCalculator.tsx`, a `"use client"` component) does live take-home estimation: salary (with a pay-period selector — annual/monthly/biweekly/weekly) or hourly income, a **before/after-taxes** toggle (post-tax skips estimation and hides the "Where you live" card), other income, filing status, and US state/city. All tax logic + data lives in `lib/taxData.ts` (`estimateTaxes()`) — real 2025 federal brackets/standard deductions + FICA, and **simplified flat per-state approximations** (clearly labeled estimates, easy to swap for full brackets). Edit tax numbers there, not in the component. The plain-language budget verdict ("Whoa there…") and per-category "running high" flags come from `lib/budgetInsights.ts` (`getBudgetInsights()`), which compares each expense to rough benchmark shares of take-home — tune the thresholds there.

**Persistence / "pick up where you left off":** `lib/storage.ts` is an SSR-safe localStorage layer (`loadJSON`/`saveJSON`/`removeStored`, keyed by versioned `STORAGE_KEYS`). The quiz saves its result on reaching the results screen and offers a "Welcome back" resume on the welcome screen; the budget calculator hydrates from and persists to storage on every change (with a "Clear" control). This is intentionally the on-ramp for real accounts — see `docs/auth-plan.md` for the **deferred Supabase auth plan** (accounts stay optional, never gate content; localStorage snapshots sync into the account on first login). Don't add forced sign-up — the brand promises "no sign-up walls."

Learn-page copy (topic headlines, intros, related topics, tool cross-links) lives in `lib/learnContent.ts`, keyed by `TopicId`. `app/learn/[topic]/page.tsx` is the shared topic-hub template.

**Articles** (157 total; ~15 per topic, Budgeting has 26, Investing 31, Insurance 5, Protecting Your Money 9) live in `lib/articles/<topic>.ts` as `Article[]` (schema in `lib/articles/types.ts`), aggregated by `lib/articles/index.ts` — add a topic file to the imports + `topicArticleSets` array there. Each topic's articles can span multiple files (e.g. `credit.ts` + `creditExtra.ts`); they merge by `topicId` regardless of file split. Article bodies are structured `ArticleBlock[]` (`p`/`h2`/`h3`/`list`/`steps`/`tip`/`key`), rendered by `components/ArticleBody.tsx` at `/learn/[topic]/[article]` (SSG). The topic hub derives its Start-here/Go-deeper/Advanced roadmap from these articles via `getTopicRoadmap()` (falls back to the optional `learnContent.path` "Coming soon" only if a topic has zero articles; `guideCount()` likewise counts real articles first). **Edit article prose in `lib/articles/<topic>.ts`, not the page.** Voice is NerdWallet-style (see `budgeting.ts` as the reference). Inline markdown in `text`: `**bold**` and `*italic*` only — no other markdown (links/headers are structural blocks). **State concrete current figures, not "check the current rate."** Articles name the actual dollar amounts (contribution limits, standard deduction, tax brackets, EITC/education-credit amounts, Pell max, federal loan limits/rates, etc.), tagged with the year (e.g. "For 2026, …") and a light "confirm at IRS.gov / it adjusts yearly" caveat — the owner accepts the yearly-update burden. Current values are 2026 (IRS retirement/tax figures), 2026 tax year (brackets/deduction/EITC), and 2025–26 (Pell $7,395, undergrad Direct Loan 6.39%). When figures genuinely vary by person/state/lender (your card's APR, insurance quotes, mortgage credit minimums, SNAP limits), give a representative anchor *and* keep the "check yours" guidance.

The article page is built for engagement: `components/ReadingProgress.tsx` (top scroll bar tracking `#article-content`), `components/ArticleToc.tsx` (scroll-spy ToC, pushed into the left margin on `xl` via the page's grid while the article column stays centered), and a drop cap on the first paragraph. Heading ids for the ToC come from `lib/articles/headings.ts` — `ArticleBody` and `extractHeadings` both call `makeHeadingIder()` in block order so the ids match. Glossary terms render as `components/GlossaryTerm.tsx` (a client popover showing the definition inline on hover/tap) instead of plain links — keeps readers on the page.

Article reading style: the **body is a reading serif** (Newsreader, `--font-reading`, applied via the `.article-body` / `.article-lead` classes in `globals.css`); headings/pull-quotes stay Fraunces (`font-display`). Each article is colored by its **topic accent** (`meta.color`): the page passes `accent` to `ArticleBody`/`ArticleToc`/`ReadingProgress` and sets `--article-accent` on `#article-content` (the drop cap reads it). That accent drives the heading marker rules, list bullets, step numbers, pull-quote border, emblem, and dot-grid graphic — so budgeting reads green, college gold, home-ownership terracotta, etc. Body text is intentionally near-black (high contrast) — don't reintroduce `/70`–`/80` opacity on article prose.

Each article's "try it yourself" tool link comes from `lib/articleTools.ts` (keyed by article slug), falling back to the topic-level `learnContent[topic].tool`.

**Cross-calculator import:** `lib/calcImports.ts` reads another calculator's saved localStorage snapshot and returns the one useful number — `readDebtSummary()` (total monthly debt payment) and `readBudgetSummary()` (take-home / expenses / leftover, recomputed from the budget snapshot via `estimateTaxes`). Targets check availability on mount and show an import affordance: the Budget Planner imports debts as a "Debt payments" expense; the Debt Payoff and Savings Goal calculators import the budget's monthly leftover (as extra payment / monthly contribution). Imports are one-time on click (the value becomes an editable field).

**Glossary:** `lib/glossary.ts` holds terms + plain-English definitions. `ArticleBody` auto-links the FIRST occurrence of any glossary term/alias in each article to `/glossary#<slug>` (with a hover definition) — authors just write normal prose using standard terms, no manual tagging. `app/glossary/page.tsx` is the A–Z glossary. To make a new term link everywhere, add it to `lib/glossary.ts`.

### The quiz (the only real logic in the app)

The quiz is the centerpiece. Its full product spec lives in `docs/quiz-spec.md` — **read it before changing quiz behavior, and update it in the same change**, since it's the source of truth for question wording, scoring rules, and results copy.

Data and logic are fully separated from rendering:

- **`lib/topics.ts`** — the 9 canonical financial topics (`TopicId` union, icons, colors, `/learn/*` hrefs). Adding a `TopicId` is a spine change: TypeScript forces matching entries in `learnContent`, and in `quizData.ts`'s `knowledgeCheckBank` / `topicWhyBlurbs` / `topicResourceCards` records (plus an optional Q3 option). Everything driven by `topics.map()` — homepage cards, Learn hub, nav dropdowns, static params — updates automatically. `getTopic(id)` throws on unknown ids. This list is the spine: homepage topic cards, quiz topics, knowledge-check banks, and resource cards all key off `TopicId`.
- **`lib/quizData.ts`** — all quiz content + pure scoring/results functions: question definitions (`quizQuestions`), the knowledge-check question bank (`knowledgeCheckBank`, keyed by `TopicId` → `Tier`), and result generators (`getFinancialProfile`, `getKnowledgeCheckFeedback`, `getWhereToStart`, `scoreKnowledgeCheck`, resource-card maps). Keep this file logic-only and presentation-free.
- **`components/QuizFlow.tsx`** (`"use client"`) — the state machine: `welcome → q1..q5 → kc → results`. Holds all answer state and drives navigation/progress. The knowledge check (`kc`) is built dynamically from Q3-selected topics at the difficulty `Tier` derived from Q5 (`confidenceTier`), pulling `KC_QUESTIONS_PER_TOPIC` (currently 2) questions per topic.
- **`components/QuizResults.tsx`** — pure presentation; takes computed answers/tier and calls the `lib/quizData.ts` functions to render the 4-part results page.

Key branching rules baked into the flow (see spec for rationale):
- **`not-sure`** on Q3 is **combinable** with real topics (plain toggle, not exclusive). If real topics are also chosen, the knowledge check runs on those topics as normal. If *only* `not-sure` is chosen, a short **general knowledge check** runs instead (`GENERAL_KC_PICKS` in `quizData.ts` — one beginner question each from a few foundational topics) and results use "let's figure it out together" framing with all topics for guidance. The knowledge check (topic or general) is also **skippable** mid-flow via a footer ("Skip this question" / "Skip to my results"); skipping sets `skippedKc`, which suppresses score feedback on the results page.
- Q5 confidence maps to a `Tier` (`beginner`/`advanced`) that selects which knowledge-check questions are asked *and* which resource-card variant is shown.
- The **confidence-mismatch** feedback messages (said-confident-but-scored-low, said-beginner-but-scored-well) are intentional and should not be removed — they're what makes the quiz feel honest.

## Styling conventions

- Tailwind v4 with the theme defined in `app/globals.css` via `@theme` (not a `tailwind.config`). Brand colors are CSS custom properties exposed as utilities. The palette is deliberately restrained — one confident green plus warm accents, **not** a rainbow:
  - `ink` (`#11211c`, near-black pine) — headings and body text. (It is no longer the background of dark *sections* — those are now `forest`; see below.)
  - `forest` (`#0c4a39`, deep green) is **the** brand green and the color of all dark chrome: the sticky header (`components/Header.tsx`, `bg-forest/95`), the footer (`components/Footer.tsx`), **every dark section/card on the homepage** (stats band, mission quote card, quiz promo, tools section, community band gradient, newsletter — all `bg-forest` / `from-forest…`), and the single green across all calculators (charts, callouts, positive numbers). Raised surfaces *on* green (cards, inputs) use `forest-700` (`#15624b`); hairlines on green use `border-white/10`. `emerald` (`#1f9069`) remains in the theme for incidental accents but was retired from the calculators so the green reads as one color. If you need a second green in a chart, pair forest with a non-green (stone/amber/terracotta), as the donuts do.
  - `amber` (`#e7a33c`) — primary CTAs/highlights — with `amber-deep` for hover/pressed.
  - `terracotta` — energetic accent, used sparingly (eyebrow dots, etc.).
  - `paper` (`#f7f2e8`, warm cream) page background, `paper-deep` for alternating sections, `sand` for warm borders/dividers, `cream` (near-white) for cards and text-on-dark, `stone` for muted secondary text on light.
- **Design direction is "warm editorial," modeled on mission-driven nonprofits (Close the Gap, SEO-USA) and NerdWallet** — real photography of real people, asymmetric layouts, confident serif headlines, honest impact stats. Avoid the old "flat AI" tells: abstract blob SVGs, evenly-spaced rainbow cards, perfectly symmetric grids.
- **Typography:** headlines use the Fraunces serif via the `font-display` class (defined in `globals.css`, backed by `--font-logo` from `app/layout.tsx`). Body/UI uses Geist sans (the default). Reach for `font-display` on any hero/section headline.
- **Imagery:** real photos live in `public/images/` (sourced from Unsplash; free license), used for **people/hero** shots — rendered with `next/image` (`fill` inside aspect-ratio containers). Portraits `portrait-1..4`, plus `students-collab`, `community`, `graduate`, `classroom`, `studying`, `mentor`. **Topic cards use a cohesive set of hand-coded flat SVG illustrations** at `public/images/illustrations/<topic>.png` (one per `TopicId`, referenced by the `image` field on each topic in `lib/topics.ts`; shown on the homepage topic grid, `/learn` hub, topic-hub heroes, and "Keep going" cards). They're generated by `scripts/build-illustrations.mjs` — a shared cream/halo/hill backdrop + brand palette + one iconic subject each, so the whole set stays consistent. Edit subjects/palette there and re-render (headless-Chrome screenshot of the SVG → PNG; command in the script header). This replaced an earlier mismatched illustration set.
- **No "AI-template" tells:** the design deliberately avoids the generic AI-site signatures — no "eyebrow pill" capsule labels (use a plain `text-xs uppercase tracking-[0.2em]` kicker instead), no Lucide-icon-in-colored-rounded-square tiles on cards, no decorative watermark icons or gradient hero washes, no emoji in UI copy. Lead with typography, whitespace, the illustrations, and real photos.
- A faint SVG paper-grain overlay is applied via `body::before` in `globals.css` to avoid a flat vector look; real content sits above it via `body > * { z-index: 2 }`. **Gotcha:** that `body > *` rule is *unlayered* CSS, so it beats Tailwind's layered `.fixed`/`.z-*` utilities (cascade layers lose to unlayered regardless of specificity). Any direct child of `<body>` — including a `createPortal` target like `components/MobileNav.tsx`'s drawer or a future modal — must set `position`/`z-index` via **inline style**, not utility classes, or it gets pinned to `position: relative; z-index: 2`.
- **Primary nav** lives in `lib/nav.ts` (the shared `nav` array, built from `topics` + `toolCategories`) — used by both the desktop header and the mobile drawer, so adding a topic/category updates both.
- **Header dropdowns:** `components/Header.tsx` (server component) renders entries with `items` as hover/focus dropdown menus (Learn → the 9 topics, Tools → the 4 categories, Start Here / Resources / About → sub-links). It's **CSS-only** (`group-hover` + `group-focus-within`, no JS) — the panel uses a `pt-3` bridge so hover isn't lost crossing the gap, and is `absolute` (not `fixed`) so the header's `backdrop-filter` containing block doesn't break it.
- **Mobile nav:** the header nav is desktop-only (`hidden lg:flex`); `components/MobileNav.tsx` (`"use client"`) is the hamburger + full-screen drawer below `lg`, portalled to `<body>` (to escape the header's `backdrop-filter` containing block), with scroll-lock, Escape-to-close, and **expandable sections** mirroring the desktop dropdowns (tap a label to navigate, tap its chevron to expand the sub-items). The drawer scrolls (`overflow-y-auto`) with the "Take the Quiz" CTA pinned.
- **Site search (⌘K):** `components/SearchDialog.tsx` (`"use client"`) is a command-palette search in the header (visible on desktop + mobile; opens on click or ⌘K/Ctrl+K). It's fully client-side over a **lightweight index built server-side** by `lib/search.ts` `getSearchItems()` — which maps articles → title/dek/href only (no bodies), plus calculators (`toolsRegistry`), topics, glossary terms, and key pages. The server `Header` calls `getSearchItems()` and passes the array as a prop, so the client bundle never imports `lib/articles`. It's a **full-width drop-down panel** (NerdWallet style): anchored directly under the sticky header (its `top` is measured from `document.querySelector("header")`'s height on open, so it tracks the header) and spanning the page width, with the search field, results, and footer in a centered `max-w-3xl` column over a full-bleed `bg-cream`. A `bg-ink/40` scrim dims everything below the header — but there is **deliberately no scroll-lock**: the scrim doesn't capture wheel events, so the page keeps scrolling behind the panel (the panel stays fixed under the header). Results are grouped into sections by kind (Topics/Calculators/Guides/Glossary/Pages — section order follows relevance, since groups are emitted in first-appearance order of the scored list) with a per-kind icon + color chip, the active row highlighted with an enter affordance, and a keyboard-hint footer. Empty state shows a "Jump to" group (topics + key pages). Portalled to `<body>` (scrim + panel) with **inline** `position:fixed` + `zIndex` 69/70 (the unlayered `body > *` rule would otherwise pin them — same gotcha as MobileNav); closes on Escape, an outside `mousedown`, or a scrim click; arrow/Enter/Esc keyboard nav. To make something findable, just ensure it's in `getSearchItems()`.
- **"Money Guide" chatbot:** `components/ChatLauncher.tsx` (`"use client"`) is a floating "Ask" launcher → chat panel, mounted once globally in `app/layout.tsx` (server, passing `getSearchItems()`). It's a **hybrid**: today it's retrieval-only via `lib/guide.ts` `guideAnswer()` (soft, stopword-stripped matching over the same index — it never invents answers, just replies with a friendly line + the best matching guides/tools/resources, with a safety nudge to Resources on sensitive queries). The seam to **flip on real AI** is the `AI_ENDPOINT` constant in `ChatLauncher.tsx`. The backend is already sketched at **`app/api/chat/route.ts`** — a Node route handler that retrieves the top matches via `rankItems`, asks Claude (Haiku by default) to answer **using only that context** (guardrails: short, no individualized legal/tax/immigration advice, no invented numbers, routes to free help), and returns `{reply, sourceHrefs[]}`. It's **inert until `ANTHROPIC_API_KEY` is set** (returns 503 → client falls back to the guide). Turn-on is 3 steps in **`docs/ai-chat-setup.md`** (get key → set env var → set `AI_ENDPOINT = "/api/chat"`). If the API errors/caps out, the chat **falls back to the retrieval guide** so it never breaks. Needs a server host (Vercel/Node), not a pure static export. Portalled to `<body>` with inline `zIndex:55` (below the mobile drawer/search), non-modal (closes on Esc/outside-click, doesn't lock the page). z-index ladder: search 70 > mobile drawer 60 > chat 55 > sticky header 50.
- Use the `@/*` import alias (maps to repo root) for cross-module imports, e.g. `@/components/Header`, `@/lib/topics`.
- Icons come from `lucide-react`. Topic/tool objects carry their `icon` as a component reference and render it as `<topic.icon ... />`. Per-topic accent colors in `lib/topics.ts` are coordinated greens/amber/terracotta (inline hex), not clashing hues.
- Pages compose `<Header />` + content + `<Footer />` inside a `min-h-screen bg-paper text-ink` wrapper. Only interactive components are `"use client"`; pages stay server components.
